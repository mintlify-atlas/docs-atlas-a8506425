## Overview

The **UX domain** provides actionable UX guidelines with do's, don'ts, code examples (good and bad), and severity ratings. Covers accessibility, navigation, animation, forms, and more.

**Database:** `ux-guidelines.csv` (99 guidelines)  
**Search columns:** Category, Issue, Description, Platform  
**Output columns:** Category, Issue, Platform, Description, Do, Don't, Code Example Good, Code Example Bad, Severity

## When to Use

- Checking accessibility compliance
- Reviewing code for UX issues
- Understanding best practices
- Fixing usability problems
- Learning do's and don'ts

## Priority by Severity

### High Severity (MUST FIX)

<Warning>
  These issues cause **legal compliance problems**, **critical usability failures**, or **accessibility violations**. Fix immediately.
</Warning>

**Examples:**
- Color contrast below 4.5:1
- Missing alt text on images
- No keyboard navigation support
- Broken back button
- Touch targets below 44x44px
- Ignoring `prefers-reduced-motion`

### Medium Severity (SHOULD FIX)

<Info>
  These issues cause **moderate usability problems** or **unprofessional appearance**. Fix before launch.
</Info>

**Examples:**
- No active nav state
- Animation durations too long
- Missing loading states
- No sticky navigation
- Line height too tight

### Low Severity (NICE TO FIX)

<Tip>
  These issues are **polish items** that improve user experience. Fix when time permits.
</Tip>

**Examples:**
- Missing breadcrumbs on shallow sites
- No smooth scroll (instant is OK)
- Basic hover effects

## Code Examples by Category

### Accessibility: Color Contrast

<CodeGroup>
```css Good
/* WCAG AA: 4.5:1 minimum */
.text {
  color: #1E293B; /* Dark slate */
  background: #F8FAFC; /* Light background */
  /* Contrast ratio: 14.1:1 ✓ */
}
```

```css Bad
/* Insufficient contrast */
.text {
  color: #94A3B8; /* Light slate */
  background: #F8FAFC; /* Light background */
  /* Contrast ratio: 2.1:1 ✗ */
}
```
</CodeGroup>

### Accessibility: Focus States

<CodeGroup>
```css Good
/* Visible focus ring */
button:focus-visible {
  outline: 3px solid #2563EB;
  outline-offset: 2px;
}
```

```css Bad
/* No focus indicator */
button:focus {
  outline: none; /* ✗ Never do this */
}
```
</CodeGroup>

### Animation: Reduced Motion

<CodeGroup>
```css Good
/* Respect user preference */
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

```css Bad
/* Ignore accessibility */
.element {
  animation: spin 2s infinite;
  /* No reduced motion check ✗ */
}
```
</CodeGroup>

### Touch: Target Size

<CodeGroup>
```html Good
<!-- 44x44px minimum (Apple HIG) -->
<button class="h-12 w-12 p-3">
  <Icon size={24} />
</button>
```

```html Bad
<!-- Too small for touch -->
<button class="h-6 w-6 p-1">
  <Icon size={16} />
</button>
```
</CodeGroup>

### Forms: Error Messages

<CodeGroup>
```jsx Good
// Clear error near input
<div>
  <input
    type="email"
    className={error ? 'border-red-500' : ''}
    aria-invalid={!!error}
    aria-describedby="email-error"
  />
  {error && (
    <p id="email-error" className="text-red-500 text-sm mt-1">
      {error}
    </p>
  )}
</div>
```

```jsx Bad
// Generic error at top
{error && <p className="text-red-500">Invalid input</p>}
<input type="email" />
<input type="password" />
<input type="text" />
// ✗ User can't tell which field is wrong
```
</CodeGroup>

### Navigation: Active State

<CodeGroup>
```jsx Good
// Visual indication of current page
<nav>
  <Link
    href="/dashboard"
    className={pathname === '/dashboard' ? 'text-primary border-b-2' : 'text-muted'}
  >
    Dashboard
  </Link>
</nav>
```

```jsx Bad
// All links look the same
<nav>
  <Link href="/dashboard" className="text-muted">
    Dashboard
  </Link>
</nav>
```
</CodeGroup>

## WCAG Compliance Checklist

<AccordionGroup>
  <Accordion title="Level A (Minimum)">
    - [ ] All images have `alt` text
    - [ ] All form inputs have labels
    - [ ] Color is not the only indicator
    - [ ] Keyboard navigation works
    - [ ] No keyboard traps
  </Accordion>
  
  <Accordion title="Level AA (Required)">
    - [ ] Color contrast 4.5:1 for normal text
    - [ ] Color contrast 3:1 for large text (24px+)
    - [ ] Focus visible on all interactive elements
    - [ ] Page has a unique `<title>`
    - [ ] Headings in correct order (h1 → h2 → h3)
    - [ ] Touch targets minimum 44x44px
  </Accordion>
  
  <Accordion title="Level AAA (Enhanced)">
    - [ ] Color contrast 7:1 for normal text
    - [ ] Color contrast 4.5:1 for large text
    - [ ] Line height 1.5+ for body text
    - [ ] Paragraph spacing 1.5x line height
    - [ ] Text resizable up to 200% without loss of content
  </Accordion>
</AccordionGroup>

## Common UX Mistakes

<Tabs>
  <Tab title="Navigation">
    ❌ **Sticky nav without padding compensation**
    ```html
    <nav class="fixed top-0">...</nav>
    <main><!-- Content hidden behind nav ✗ --></main>
    ```
    
    ✅ **Correct approach**
    ```html
    <nav class="fixed top-0 h-16">...</nav>
    <main class="pt-16"><!-- Content visible ✓ --></main>
    ```
  </Tab>
  
  <Tab title="Animation">
    ❌ **Too many animations**
    ```html
    <div class="animate-bounce">
      <div class="animate-pulse">
        <div class="animate-spin">✗</div>
      </div>
    </div>
    ```
    
    ✅ **Minimal, purposeful animation**
    ```html
    <button class="hover:scale-105 transition-transform">✓</button>
    ```
  </Tab>
  
  <Tab title="Forms">
    ❌ **Too many fields**
    ```html
    <!-- 10+ fields = low conversion ✗ -->
    <input name="firstName" />
    <input name="lastName" />
    <input name="email" />
    <input name="phone" />
    <input name="address" />
    ...
    ```
    
    ✅ **Minimal fields**
    ```html
    <!-- 3 fields max = high conversion ✓ -->
    <input name="email" />
    <input name="password" />
    <button>Sign Up</button>
    ```
  </Tab>
  
  <Tab title="Touch">
    ❌ **Hover-only interactions on mobile**
    ```css
    /* Mobile users can't hover ✗ */
    .menu { display: none; }
    .nav:hover .menu { display: block; }
    ```
    
    ✅ **Click/tap interactions**
    ```jsx
    // Works on mobile ✓
    const [open, setOpen] = useState(false);
    <button onClick={() => setOpen(!open)}>Menu</button>
    ```
  </Tab>
</Tabs>

## Testing Checklist

### Before Launch

<Steps>
  <Step title="Keyboard Navigation">
    - Tab through entire page
    - Verify focus visible on all elements
    - Check tab order matches visual order
    - Test Enter/Space on buttons
  </Step>
  
  <Step title="Screen Reader">
    - Test with VoiceOver (Mac) or NVDA (Windows)
    - Verify all images have alt text
    - Check form labels announced correctly
    - Ensure page title read aloud
  </Step>
  
  <Step title="Color Contrast">
    - Use browser DevTools "Accessibility" panel
    - Check all text meets 4.5:1 minimum
    - Verify CTAs have 7:1+ contrast
  </Step>
  
  <Step title="Responsive">
    - Test at 375px (mobile), 768px (tablet), 1024px (laptop)
    - Verify no horizontal scroll
    - Check touch targets 44x44px minimum
    - Test sticky elements don't obscure content
  </Step>
  
  <Step title="Performance">
    - Run Lighthouse in Chrome DevTools
    - Target: Accessibility score 95+
    - Check `prefers-reduced-motion` respected
  </Step>
</Steps>
