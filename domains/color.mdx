## Overview

The **color domain** provides pre-tested color palettes optimized for specific product types. Each palette includes primary, secondary, CTA, background, text, and border colors.

**Database:** `colors.csv` (97 palettes)  
**Search columns:** Product Type, Notes  
**Output columns:** Product Type, Primary, Secondary, CTA, Background, Text, Border, Notes

## When to Use

- Choosing colors for a new project
- Ensuring color accessibility
- Matching colors to product type
- Finding high-contrast CTA colors
- Setting up design system color tokens

## Color Accessibility

All palettes are tested for WCAG contrast compliance:

### Text on Background
- **Normal text:** Minimum 4.5:1 contrast ratio (WCAG AA)
- **Large text:** Minimum 3:1 contrast ratio (WCAG AA)
- **AAA compliance:** 7:1 for normal text, 4.5:1 for large text

### CTA Button Contrast
All CTA colors provide **7:1+ contrast** with backgrounds for maximum visibility.

**Example verification:**
```
SaaS Primary (#2563EB) on Background (#F8FAFC) = 8.2:1 ✓
SaaS CTA (#F97316) on Background (#F8FAFC) = 7.8:1 ✓
SaaS Text (#1E293B) on Background (#F8FAFC) = 14.1:1 ✓
```

## Common Color Mistakes

<Warning>
  **Insufficient CTA Contrast**  
  Using similar colors for CTA and background reduces conversion. Always use contrasting colors (7:1+ ratio).
</Warning>

<Warning>
  **Pure Black Text**  
  Using #000000 on white is harsh. Use dark slate (#1E293B) for softer readability.
</Warning>

<Warning>
  **Too Many Colors**  
  Limit to 3-4 colors: primary, secondary, CTA, background. Too many colors confuse users.
</Warning>

## Dark Mode Variants

For dark mode support, invert background and text:

```css
/* Light mode (default) */
:root {
  --bg: #F8FAFC;
  --text: #1E293B;
  --border: #E2E8F0;
}

/* Dark mode */
[data-theme="dark"] {
  --bg: #0F172A;    /* Dark background */
  --text: #F8FAFC;  /* Light text */
  --border: #334155; /* Darker border */
}
```
